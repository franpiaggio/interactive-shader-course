Raymarching for Creative Art · Lesson 06 · Building scenes

Composing a scene

A scene is one function — map(vec3 p) — that returns the distance to the nearest surface. Build it from 3D primitives, transforms, and the boolean operators you already know from 2D. They transfer with zero changes.

Your win this lesson Write a map() that combines several primitives — a ground plane, a spinning box, a smin-merged blob — into one composed scene.

3D primitives — same as 2D, plus one axis

float sdSphere(vec3 p, float r) { return length(p) - r; }
float sdBox(vec3 p, vec3 b)     { vec3 d = abs(p) - b;
    return length(max(d,0.0)) + min(max(d.x,max(d.y,d.z)),0.0); }
float sdTorus(vec3 p, vec2 t)   { vec2 q = vec2(length(p.xz)-t.x, p.y); return length(q)-t.y; }
float sdPlane(vec3 p, float y)  { return p.y - y; }   // ground: union with min to add a floor

The booleans (min, max, smin) and transforms work exactly as in Lesson 03 — only p is a vec3 now. To move a shape, subtract a position; to rotate, multiply p by a rotation matrix before the SDF call:

mat3 rotY(float a){ float c=cos(a),s=sin(a); return mat3(c,0,s, 0,1,0, -s,0,c); }
// ... sdBox(rotY(iTime) * p, vec3(0.5))   spins the box around the vertical axis

The map() convention

float map(vec3 p) {
    float ground = sdPlane(p, -1.0);
    float blob   = smin(sdSphere(p, 0.8), sdBox(p - vec3(1.0,0,0), vec3(0.4)), 0.3);
    return min(ground, blob);     // floor unioned with the merged blob
}

Play — compose with sliders

Ground + sphere + box + torus, all in one map(). blend melts the sphere and box together; box x slides the box through the sphere; spin rotates the whole group. This is scene-building as a live instrument.

Now you — build the map(), graded live

Two tasks: spin the box, and give the scene a floor. Check grades each.

Watch for Transform order: sdBox(rotY(a) * (p - pos), b) rotates then the subtraction placed it — apply the translation (p - pos) and rotation in the order you actually want, or the shape orbits instead of spinning in place.
