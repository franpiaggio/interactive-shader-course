Raymarching for Creative Art · Lesson 09 · Infinite worlds

Domain repetition

This is the trick that makes raymarching feel like magic: tile a single shape across infinite space with one line of arithmetic on the point — no extra cost, no extra geometry.

Your win this lesson Turn one primitive into an endless lattice by folding the domain with mod — and color each repeated cell independently using its grid id.

Fold space, not shapes

You don't copy the shape. You fold the coordinate before evaluating the SDF: wrap p into a single repeating cell, so every cell looks identical to the one at the origin:

vec3 q = mod(p + 0.5*s, s) - 0.5*s;   // wrap p into a cell of size s, centered
return sdSphere(q, r);                  // one sphere → an infinite grid of spheres

The whole 3D space now tiles with copies, spaced s apart, for the price of one mod. Repeat on only some axes (p.xz) for an infinite floor of objects; all three for a crystal.

Per-cell variation The cell id round(p/s) is constant within a cell — feed it to a hash and a palette and every copy gets its own color or height, so the lattice looks designed, not stamped.

Play — an endless lattice

spacing packs the cells tighter or looser; radius sizes each shape. Each cell is tinted by its id, so you're flying through a colored crystal. Drag to orbit.

Now you — tile it, graded live

One lonely sphere. Fold the domain so it tiles across the whole frame. Check counts how much of the screen is now covered by repeated copies.

Watch for Plain mod repetition assumes the shape fits inside its cell — if r approaches s/2, neighbors touch and the SDF stops being exact near the seams. Keep shapes comfortably smaller than the spacing, or use IQ's limited repetition for a finite count.
