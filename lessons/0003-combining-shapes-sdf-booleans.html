Combining shapes: SDF booleans

One shape is a primitive. Art is shapes combined. With signed distance fields, union, subtraction, and a buttery smooth merge are each a single line — and they transfer to 3D completely unchanged.

Your win this lesson Combine two SDFs with min (union) and blend them organically with smin (smooth union) — the single most useful operator in procedural art.

The boolean trio

Because an SDF returns distance with a sign, set operations are just arithmetic on the distances:

float opUnion    (float a, float b) { return min(a, b); }   // both shapes
float opIntersect(float a, float b) { return max(a, b); }   // overlap only
float opSubtract (float a, float b) { return max(a, -b); }  // a with b carved out

Take the closest surface (min) and you're inside if you're inside either shape: that's a union. The other two are the same idea with flipped signs.

The star: smooth minimum

A plain min leaves a sharp crease where shapes meet. smin blends the two fields near the join, melting them together like clay — this is what makes raymarched scenes look organic instead of CSG-blocky:

float smin(float a, float b, float k) {
    float h = clamp(0.5 + 0.5*(b - a)/k, 0.0, 1.0);
    return mix(b, a, h) - k*h*(1.0 - h);
}

k is the blend radius: 0 = a hard min, larger = more melt.

This is a 3D skill in disguise Everything here — min, max, smin — works identically when the point is a vec3. The scene function you'll write for raymarching (map(vec3 p)) is exactly this: primitives combined with these operators. You're writing it in 2D first so the geometry is easy to see.

Play — melt two shapes

A circle and a box stacked vertically. blend k melts them together; gap slides them apart. Watch the bridge of "material" form between them as k rises.

Now you — combine them, graded live

The starter draws only the circle. Two tasks: union the box in, then make the merge smooth so the gap between them fills. Check grades each by reading the pixels.

Watch for Sign matters: opSubtract is max(a, -b) — forget the minus and you get an intersection instead of a cut. And smin with k = 0 is just min; you need a positive k to see any melt.
