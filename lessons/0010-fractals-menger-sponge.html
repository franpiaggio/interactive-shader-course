Raymarching for Creative Art · Lesson 10 · Infinite detail

Fractals: the Menger sponge

This is what raymarching does that nothing else can touch: infinite, self-similar detail from a few lines of math. A loop that carves the same hole at ever-smaller scales builds a structure no mesh could ever store.

Your win this lesson Understand and run an iterated SDF — the Menger sponge — and control its recursion depth, the moment procedural detail becomes effectively free.

Self-similarity by iteration

You already know half of this A fractal is just two tools you've met, stacked: fold the domain (Lesson 09's mod, so one carve repeats everywhere) and subtract (Lesson 03's max, to cut the hole). The only new move is doing it in a loop that shrinks the scale each pass. Don't try to picture all of it at once — picture one carve, then trust the loop.

Start with a box. Carve a cross-shaped hole through it (that's the "+1" Menger step). Now do the same carve again on each remaining piece, three times smaller — and again, and again. Each pass multiplies the detail; the SDF stays a dozen lines:

float menger(vec3 p) {
    float d = sdBox(p, vec3(1.0));
    float s = 1.0;
    for (int m = 0; m < ITER; m++) {
        vec3 a = mod(p*s, 2.0) - 1.0;
        s *= 3.0;
        vec3 r = abs(1.0 - 3.0*abs(a));
        float c = (min(min(max(r.x,r.y), max(r.y,r.z)), max(r.z,r.x)) - 1.0) / s;
        d = max(d, c);               // subtract the cross at this scale
    }
    return d;
}

Each iteration adds a finer level of holes. Three or four passes already looks impossibly intricate; the cost is just a few more loop turns per ray step.

Reading the carve, one line at a time

The middle four lines look cryptic; here's each one in plain terms — you never have to memorise this, just see that nothing is magic:

  • a = mod(p*s, 2.0) - 1.0 — fold space into repeating −1…1 cells at the current scale s (exactly the Lesson 09 fold). Every cell now looks the same.
  • s *= 3.0 — the next pass works on a grid three times finer.
  • r = abs(1.0 - 3.0*abs(a)) — distance to the cross arms inside the cell. abs(a) mirrors into one quadrant; the rest measures how far you are from the cross's bars.
  • c = (min(min(max(r.x,r.y), …)) - 1.0)/s — turn that into the SDF of the cross-shaped cutter, scaled back down by s so it matches this pass's size.
  • d = max(d, c) — subtract the cutter from what we have (Lesson 03's carve). That's the hole.
Two new helpers you'll see in the lab rotX(a) and rotY(a) are rotation matrices — multiply a point by one and it spins around that axis by angle a (built from sin/cos). Rotating p before map spins the whole fractal. You'll lean on these from here on; copy them as-is for now.

Watch for Iterated SDFs are only approximate distances. The folding and s *= 3.0 scaling stretch space, so the value returned can be a little larger than the true nearest distance — and a full sphere-tracing step might then jump straight past a thin wall. If the surface looks eroded, lower the hit threshold or take a fraction of the step (t += d*0.85, as the lab does). More iterations also cost more — there's a real budget here.
