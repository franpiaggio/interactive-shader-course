Build an infinite sphere field

A capstone that wires together everything you already know — a flythrough lattice of shaded spheres — and then adds the one move that makes it feel alive: a travelling pulse of light that sweeps through the field, shrinking and lighting each sphere as it passes.

Your win this lesson Assemble a raymarched, infinite sphere field, then write the signature effect from scratch: a pulse that travels along the view, made of one line of exp.

The build, step by step

Almost every step here is something you've already shipped in an earlier lesson. The whole point of a project is assembly — so this is mostly a map back to the rungs you climbed, with two genuinely new ideas added on top (marked new).

  1. Start from a flat shader. Every pixel runs your code and gets a colour from its coordinate — Lesson 01.
  2. Describe one shape with an SDF. length(p) - r is the distance to a sphere — Lesson 02, Lesson 04.
  3. Subdivide space. Fold the point with mod / round and that one sphere tiles into an endless grid — Lesson 09.
  4. Add a camera that moves. Build rays from the four camera vectors and fly a path through the lattice — Lesson 05. new: a path that alternates weaving forward with vertical dives (see the note below).
  5. A travelling light pulse. new — the heart of this lesson, below.
  6. Per-sphere variation. new — give each cell its own size, colour and drift with a hash (the same hash from Lesson 09).
  7. Finish it. Normals + light + fog + gamma + a touch of bloom and grain — Lesson 07, Lesson 08, Lesson 12.

The signature move: a travelling pulse

We want a band of light that slides through the field — bright on the spheres it's passing, dark everywhere else, and clearly moving. The whole effect is one function: a soft bump that follows a moving plane.

float gFront;                              // where the pulse front is right now (a z position)
float pulse(vec3 c){
    float d = c.z - gFront;                // how far this sphere is from the front
    return exp(-0.25 * d*d);               // 1 at the front, fading smoothly to 0
}
// ...then, per sphere:
r   -= 0.42 * pulse(c);                     // the pulse SHRINKS the sphere
col += vec3(1.0,0.55,0.2) * pulse(c);      // ...and GLOWS it
Read the formula — why exp(-k·d²) is a bump d is the signed distance from a sphere to the pulse front. Square it and it's always positive, smallest (zero) right at the front. exp of a negative number is between 0 and 1: at the front exp(0)=1; step away and d² grows so exp(-k·d²) slides smoothly toward 0. That's a soft, symmetric hill centred on the front — a Gaussian. k is the width dial: bigger k = a tighter, sharper band. To make it travel, just move gFront over time. Subtract the bump from the radius to dent the sphere; add it to the colour to light it. Same bump, two uses, perfectly in sync.

Play — speed, width, depth of the pulse

speed moves the front faster; width is the k dial (higher = a thinner band); shrink is how much each sphere dents as the pulse hits it. Drag to orbit and watch the band sweep through.

Now you — bring the pulse to life, graded live

Here's the field with a pulse front already sweeping, but nothing reacting to it yet. Add the two lines marked C1 so the spheres shrink and glow as the front passes. Check watches a few pixels over time and confirms a moving band of light has appeared.

The two new ideas, briefly Per-sphere variation: the cell id round(p/2) fed through a hash gives each sphere a stable random number — use it to vary radius, tint, and a tiny per-cell sin(iTime + phase) orbit, so the field reads as alive rather than stamped (exactly the per-cell trick from Lesson 09). The exploring camera: it alternates legs — weaving forward, then turning to dive up or down — using smoothstep to ease between them. Because every sphere centre sits on odd coordinates, keeping one camera axis on an even number guarantees the path always threads the gaps and never ends up inside a sphere.
Watch for When spheres vary in size or drift, neighbouring cells differ, and a plain mod march can overshoot and slice through a sphere. The cheap fix used here is a conservative step — t += d * 0.6 — so the ray never strides past a surface a different cell was hiding.

Ship it

This compact, self-contained version pastes straight into a new Shadertoy (and runs in any of this course's labs). It's the whole idea in ~30 lines: infinite field, travelling pulse, per-cell tint, orbit camera.

float hash(vec3 p){ p=fract(p*0.3183099+0.1); p*=17.0; return fract(p.x*p.y*p.z*(p.x+p.y+p.z)); }
float gFront;
float pulse(vec3 c){ float d=c.z-gFront; return exp(-0.25*d*d); }
float map(vec3 p){
    vec3 c = 2.0*round(p/2.0);             // a sphere at every even lattice point
    float r = 0.62 - 0.42*pulse(c);        // the pulse shrinks it
    return length(p-c) - r;
}
float march(vec3 ro, vec3 rd){ float t=0.0;
    for(int i=0;i<140;i++){ float d=map(ro+rd*t)*0.6; if(d<0.001) return t; if(t>60.0) break; t+=d; }
    return -1.0; }
vec3 nrm(vec3 p){ vec2 e=vec2(1.0,-1.0)*0.5773*0.0015;
    return normalize(e.xyy*map(p+e.xyy)+e.yyx*map(p+e.yyx)+e.yxy*map(p+e.yxy)+e.xxx*map(p+e.xxx)); }

void mainImage(out vec4 fragColor, in vec2 fragCoord){
    vec2 uv=(fragCoord-0.5*iResolution.xy)/iResolution.y;
    gFront = -6.0 + mod(iTime*2.5, 12.0);                  // the front sweeps in z
    float ang = iMouse.z>0.0 ? (iMouse.x/iResolution.x)*6.28 : iTime*0.2;
    vec3 ro = vec3(6.0*sin(ang), 3.0, 6.0*cos(ang));
    vec3 ww=normalize(-ro), uu=normalize(cross(ww,vec3(0,1,0))), vv=cross(uu,ww);
    vec3 rd=normalize(mat3(uu,vv,ww)*vec3(uv,1.4));
    vec3 col=vec3(0.03,0.04,0.07);
    float t=march(ro,rd);
    if(t>0.0){
        vec3 p=ro+rd*t; vec3 c=2.0*round(p/2.0); vec3 n=nrm(p);
        float dif=max(dot(n,normalize(vec3(0.6,0.8,0.4))),0.0);
        vec3 base=mix(vec3(0.12,0.2,0.34), vec3(0.22,0.3,0.46), hash(c));
        col=base*(0.2+dif) + vec3(1.0,0.55,0.2)*pulse(c);  // shade + pulse glow
        col=mix(col, vec3(0.03,0.04,0.07), 1.0-exp(-t*0.04));
    }
    col=pow(col, vec3(0.4545));
    fragColor=vec4(col,1.0);
}
