Raymarching for Creative Art · Lesson 11 · Volume

Volumetric clouds

Until now a ray stopped at the first surface. Clouds, smoke, and fog have no surface — so you march through them, sampling density and accumulating light step by step. Same loop, a completely different look.

Your win this lesson Raymarch a volume: step through a noise-based density field, accumulate color and opacity, and light it toward the sun — a drifting cloud you can tune.

Surfaces stop; volumes accumulate

Until now the loop asked one yes/no question — did the ray hit? A volume asks a softer one at every step — how much stuff is here? So instead of if (d < 0.001) hit, you take fixed steps and at each one read a density (0 = clear air, 1 = solid cloud). Each step is a thin slab of fog; you stack their colors front-to-back until the ray can't see through any more.

Build the intuition with the simplest volume first Forget noise for a second. Imagine density(p) just returns a constant 0.05 everywhere — a uniform haze. Stepping through it, each slab adds a little grey and a little opacity, and far things fade out: that's already fog. Clouds are the same loop — the only change is swapping that constant for fbm (a few octaves of the Lesson 09 hash stacked at finer scales), so density is lumpy instead of flat. The helper is in the lab below. 
for (int i = 0; i < STEPS; i++) {
    float d = density(p);                  // 0..1 at this point
    if (d > 0.01) {
        vec4 c = vec4(cloudColor, d);       // color + opacity of this slab
        c.rgb *= c.a;                        // premultiply
        sum += c * (1.0 - sum.a);            // front-to-back compositing
    }
    if (sum.a > 0.99) break;                 // fully opaque — stop early
    p += rd * stepSize;
}
Decoding the two compositing lines c.rgb *= c.a (premultiply): a faint slab should add only a faint bit of colour, so scale its colour by its own opacity. sum += c * (1.0 - sum.a) (front-to-back): whatever you've already accumulated (sum.a opacity) partly blocks the view, so each new slab only contributes through the gap that's still clear, 1 - sum.a. Marching front-to-back means nearer cloud correctly hides the cloud behind it — and once sum.a nears 1 there's nothing left to see, so you stop.

Cheap lighting: density difference toward the sun

A volume's "normal" is the density gradient — and you find it the same way you found the SDF normal in Lesson 07: sample at an offset and compare (a finite difference). Here you sample density one short step toward the sun: if it drops off, this point is near the lit edge of the cloud and glows; if it stays dense, sunlight had to push through more cloud to get here, so it's in shadow. One subtraction buys believable self-shadowed clouds.

Play — shape the sky

coverage grows the clouds from wisps to overcast; sun x/y moves the light, changing which edges glow. The field drifts with time. This is a postable skyscape on its own.

Now you — make it cloudy, graded live

Right now the sky is clear — coverage is zero. Raise COV until clouds fill the frame. Check verifies real (bright, opaque) cloud appears.

Watch for Step size is a quality/cost dial: too large and clouds band into slabs; too small and it crawls. And density must be clamped to 0..1 — negative density makes compositing explode.
